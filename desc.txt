Пользователь может быть неавторизован или иметь несколько способов авторизации.
"Credentials" принадлежат только одному пользователю.

Пользователь может просматривать варианты реализации песен (все данные public)

Авторизованный пользователь может давать задачи по добавлению новых песен Пауку, который парсит заданный url. И, в случае успеха добавляет новый вариант песни. 

Есть тонкий момент, песня: автор/название
Из этого - один вариант песни может принадлежать двум "песням", из-за того, что песня двух авторов и на страничке каждого хранится своя "копия".
