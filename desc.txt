Архив подборов аккордов к песням

Стоянов Дмитрий, M4139
Агапов Георгий, M4138

1. Описание
1.1 Общее описание
Система позволяет просматривать каталог имеющихся песен, просматривать имеющиеся в базе подборы аккордов к песням.
Все песни в базе публичны - доступны любому посетителю веб-приложения.

Также авторизованные пользователи могут отдавать задачи на добавление нового варианта ("подбора") песни (в том числе для еще несуществующей в базе песни).
Такая задача характеризуется адресом веб-страницы, с которой можно взять текст с подбором аккордов.

1.2 Просмотр песен
Система предоставляет интерфейс, позволяющий просмотреть список песен, выбрать интересующую, запросить для нее другой вариант подбора.

1.3 Авторизация 
Предоставляется два способа авторизации: через пару логин-пароль и с помощью авторизации через сервер VK.

1.4 Создание задачи парсеру
Авторизованный пользователь задает url, по которому парсер будет искать подбор песни.
Парсер поддерживает ограниченно количество сайтов ("форматов").

----

2. Описание предметной области
В архиве хранятся песни. Песня задается своим названием и автором.
Также в архиве хранятся "варианты песен".
Вариант песни характеризуется песней, подбор которой он задает, и текстом.

При необходимости авторизации, пользователь предоставляет необходимые данные и получает токен,
который далее может предъявить при создании новой задачи.

3. Описание целей проектируемой системы
Цель проектируемой системы предоставлять тексты имеющихся песен и позволять добавлять новые песни с помощью имеющегося парсера.

4.Описание основных функций, выполняемых системой, указание ограничений
и условий выполнения этих функций
Функции:
Просмотр каталога песен.
Просмотр основного подбора конкретной песни.
Просмотр альтернативных вариантов конкретной песни.
Авторизация через логин-пароль.
Авторизация через VK.
Создание задачи для парсера (требует авторизации).

Ограничения:
Парсер поддерживает ограниченное количество сайтов.

5. Описание критериев оценки степени достижения поставленных целей.
Гарантии:
Для каждой песни известно, кем была отдана задача на ее добавление.
(Ограничиваем возможность переполнения очереди задач)
Возможность добавления песни проверяется перед попаданием песни в очередь задач (т.е. если пользователь успешно добавил задачу на добавление, песня должна быть добавлена).
